## ADDED Requirements

### Requirement: 线上分页数据一致性

系统 MUST（必须）让本地 `http://localhost:8080/` 展示与 `https://v.flomoapp.com/mine` 通过分页加载获得的全部备忘录数据一致的核心数据，包括统计数字、置顶标签、标签列表、全部备忘录内容、备忘录标签和备忘录排序。

#### Scenario: 分页加载后全部数据一致

- **WHEN** 使用 Playwright 打开线上页面并滚动或触发分页加载直到没有更多备忘录可加载
- **THEN** 本地页面展示的统计、标签、备忘录文本、备忘录标签和排序必须与线上页面全量采集结果一致

#### Scenario: 默认数据来自后端

- **WHEN** 本地前端首次加载备忘录工作台
- **THEN** 前端必须通过 FastAPI 读取默认备忘录数据，而不是在 UI 层维护独立默认数据

### Requirement: 完整 CRUD 行为

系统 MUST（必须）支持备忘录创建、读取、更新和删除的完整闭环，并在前端交互、后端 API 和 E2E 流程中保持一致。

#### Scenario: 创建备忘录

- **WHEN** 用户在本地页面输入新备忘录并提交
- **THEN** 后端必须创建记录，前端列表必须展示新记录，E2E 测试必须能观察到该记录

#### Scenario: 更新备忘录

- **WHEN** 用户编辑已有备忘录内容并保存
- **THEN** 后端必须保存更新，前端列表必须展示更新后的内容，刷新页面后该内容仍然存在

#### Scenario: 删除备忘录

- **WHEN** 用户删除已有备忘录
- **THEN** 后端必须删除记录，前端列表必须移除该记录，刷新页面后该记录不得重新出现

### Requirement: 验证覆盖

系统 MUST（必须）用自动化验证覆盖数据一致性、后端 CRUD、前端渲染和 Docker Compose 部署入口。

#### Scenario: 自动化验证通过

- **WHEN** 执行后端单元测试、前端单元测试、E2E 测试和 Docker Compose 启动验证
- **THEN** 所有验证必须通过，且验证记录必须在 PR 或任务汇报中列出
